## üü® JavaScript Fundamentals

### **Variable Declarations**
```javascript
// ‚úÖ Use const by default
const API_URL = 'https://api.example.com';
const users = [];

// ‚úÖ Use let for reassignment
let currentUser = null;
currentUser = { id: 1, name: 'John' };

// ‚ùå Avoid var (function-scoped, hoisting issues)
var oldStyle = 'avoid this';
```

### **Function Declarations**
```javascript
// ‚úÖ Arrow functions for callbacks and short functions
const processData = (data) => data.map(item => item.value);

// ‚úÖ Function declarations for main functions (hoisted)
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// ‚úÖ Async/await over Promises
async function fetchUser(id) {
  try {
    const response = await fetch(`/api/users/${id}`);
    return await response.json();
  } catch (error) {
    console.error('Failed to fetch user:', error);
    throw error;
  }
}
```

### **Object and Array Patterns**
```javascript
// ‚úÖ Destructuring
const { name, email } = user;
const [first, second, ...rest] = array;

// ‚úÖ Spread operator
const newUser = { ...user, lastLogin: new Date() };
const allItems = [...existingItems, ...newItems];

// ‚úÖ Optional chaining
const city = user?.address?.city;
const firstItem = items?.[0];

// ‚úÖ Nullish coalescing
const displayName = user.name ?? 'Anonymous';
const port = process.env.PORT ?? 3000;
```

## üü¶ ES Modules (ESM) Rules

### **Basic Import/Export Syntax**
```javascript
// ‚úÖ Named exports
export const CONFIG = { port: 3000 };
export function helper() { /* ... */ }
export class MyClass { /* ... */ }

// ‚úÖ Default exports
export default class Application {
  // Main class
}

// ‚úÖ Named imports
import { CONFIG, helper } from './utils.js';

// ‚úÖ Default imports
import Application from './App.js';

// ‚úÖ Mixed imports
import Application, { CONFIG, helper } from './App.js';

// ‚úÖ Namespace imports
import * as utils from './utils.js';
```

### **üö® Critical ESM Rules**

#### **1. File Extensions Are Required**
```javascript
// ‚úÖ Always include .js extension for relative imports
import { logger } from './logger.js';
import { config } from '../config.js';

// ‚ùå Will not work in ESM
import { logger } from './logger';
import { config } from '../config';

// ‚úÖ External packages don't need extensions
import express from 'express';
import { readFile } from 'fs/promises';
```

#### **2. Directory Structure Matters**
```
src/
‚îú‚îÄ‚îÄ index.js
‚îú‚îÄ‚îÄ utils/
‚îÇ   ‚îú‚îÄ‚îÄ logger.js
‚îÇ   ‚îî‚îÄ‚îÄ config.js
‚îî‚îÄ‚îÄ services/
    ‚îî‚îÄ‚îÄ api.js
```

```javascript
// From index.js
import { logger } from './utils/logger.js';     // ‚úÖ Relative path
import { ApiService } from './services/api.js'; // ‚úÖ Relative path

// From utils/logger.js  
import { config } from './config.js';           // ‚úÖ Same directory
import { ApiService } from '../services/api.js'; // ‚úÖ Up and down
```

#### **3. Top-Level Await**
```javascript
// ‚úÖ Allowed in ESM (Node.js 14.8+)
const config = await import('./config.js');
const data = await fetch('/api/data');

// ‚úÖ But wrap in async function for better error handling
async function initialize() {
  try {
    const config = await import('./config.js');
    // ... rest of initialization
  } catch (error) {
    console.error('Initialization failed:', error);
  }
}
```

#### **4. Dynamic Imports**
```javascript
// ‚úÖ Conditional imports
if (process.env.NODE_ENV === 'development') {
  const { devTools } = await import('./dev-tools.js');
  devTools.setup();
}

// ‚úÖ Lazy loading
async function loadPlugin(pluginName) {
  const plugin = await import(`./plugins/${pluginName}.js`);
  return plugin.default;
}
```

## üü© TypeScript Best Practices

### **TypeScript + ES Modules**
```typescript
// ‚úÖ Import .js extensions in TypeScript files
import { User } from './types.js';        // .js not .ts!
import { logger } from './logger.js';     // .js not .ts!
import { config } from './config.js';     // .js not .ts!

// ‚úÖ Type-only imports
import type { ApiResponse } from './api.js';
import type { Config } from './config.js';
```

```json

  "compilerOptions": {
```

### **Type Definitions**
```typescript
// ‚úÖ Interface for object shapes
interface User {
  readonly id: string;
  name: string;
  email: string;
  createdAt: Date;
}

// ‚úÖ Type for unions and primitives
type Status = 'pending' | 'approved' | 'rejected';
type UserId = string;

// ‚úÖ Generic types
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

// ‚úÖ Utility types
type PartialUser = Partial<User>;
type UserEmail = Pick<User, 'email'>;
type CreateUser = Omit<User, 'id' | 'createdAt'>;
```

### **Function Typing**
```typescript
// ‚úÖ Function signatures
function processUser(user: User): Promise<User> {
  return updateDatabase(user);
}

// ‚úÖ Arrow function typing
const validateEmail = (email: string): boolean => {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
};

// ‚úÖ Async function typing
async function fetchUsers(): Promise<User[]> {
  const response = await fetch('/api/users');
  return response.json();
}

// ‚úÖ Callback typing
function processItems<T>(
  items: T[], 
  callback: (item: T) => void
): void {
  items.forEach(callback);
}
```

## üü™ Node.js Specific Guidelines

### **Package.json Configuration**
```json
{
  "name": "my-app",
  "type": "module",                    // ‚úÖ Required for ESM
  "engines": {
    "node": ">=18.0.0"                 // ‚úÖ Specify Node version
  },
  "main": "./dist/index.js",           // ‚úÖ Compiled output
  "types": "./dist/index.d.ts",        // ‚úÖ TypeScript definitions
  "exports": {                         // ‚úÖ Modern exports
    ".": {
      "import": "./dist/index.js",
      "types": "./dist/index.d.ts"
    }
  },
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js",
    "dev": "tsc --watch"
  }
}
```

### **Environment Variables**
```javascript
// ‚úÖ Use process.env with defaults
const PORT = process.env.PORT ?? 3000;
const NODE_ENV = process.env.NODE_ENV ?? 'development';

// ‚úÖ Type environment variables in TypeScript
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      PORT?: string;
      DATABASE_URL: string;
      JWT_SECRET: string;
    }
  }
}
```

### **Error Handling**
```javascript
// ‚úÖ Async error handling
async function safeOperation() {
  try {
    const result = await riskyOperation();
    return { success: true, data: result };
  } catch (error) {
    console.error('Operation failed:', error);
    return { success: false, error: error.message };
  }
}

// ‚úÖ Process error handlers
process.on('uncaughtException', (error) => {
  console.error('Uncaught Exception:', error);
  process.exit(1);
});

process.on('unhandledRejection', (reason, promise) => {
  console.error('Unhandled Rejection at:', promise, 'reason:', reason);
});
```

## ‚ö†Ô∏è Common Pitfalls & Solutions

### **1. Module Resolution Issues**
```javascript
// ‚ùå Common mistake
import { utils } from './utils';       // Missing .js extension

// ‚úÖ Correct
import { utils } from './utils.js';    // Always .js for relative imports
```

### **2. Default vs Named Exports**
```javascript
// ‚ùå Mixed export styles in same file
export default class User {}
export const CONFIG = {};

// ‚úÖ Consistent approach - choose one
// Option A: Named exports only
export class User {}
export const CONFIG = {};

// Option B: Default export with named
export default class User {}
export { CONFIG };
```

### **3. TypeScript Import Extensions**
```typescript
// ‚ùå Wrong extension
import { User } from './types.ts';     // Don't use .ts

// ‚úÖ Correct extension  
import { User } from './types.js';     // Always .js for local imports
```

### **4. Async/Await vs Promises**
```javascript
// ‚ùå Promise chaining
function fetchData() {
  return fetch('/api')
    .then(response => response.json())
    .then(data => processData(data))
    .catch(error => handleError(error));
}

// ‚úÖ Async/await
async function fetchData() {
  try {
    const response = await fetch('/api');
    const data = await response.json();
    return processData(data);
  } catch (error) {
    handleError(error);
    throw error;
  }
}
```

### **5. Environment Configuration**
```javascript
// ‚ùå No fallbacks
const port = process.env.PORT;
const dbUrl = process.env.DATABASE_URL;

// ‚úÖ Always provide defaults/validation
const port = Number(process.env.PORT) || 3000;
const dbUrl = process.env.DATABASE_URL ?? 
  (() => { throw new Error('DATABASE_URL is required') })();
```

## üìù Final Tips

1. **Always use `.js` extensions** for relative imports in TypeScript ESM projects
2. **Be consistent** with export styles (default vs named)
3. **Use `type: "module"`** in package.json for ESM
4. **Prefer async/await** over Promise chaining
5. **Validate environment variables** with defaults
6. **Use strict TypeScript settings** for better code quality
7. **Configure ESLint and Prettier** for consistent code style
8. **Use workspace configurations** for monorepos
9. **Always handle errors** in async functions
10. **Keep dependencies up to date** but pin versions for stability
